# Quick Task 260704-o31: Enable Tauri devtools for debugging Summary

**Enabled the `devtools` Cargo feature on the `tauri` dependency so DevTools works in the release build — a temporary aid to diagnose why the packaged desktop app shows "Tidak dapat terhubung ke server" on login despite the backend URL, CSP, and CORS all checking out correctly from the outside.**

## Performance

- **Duration:** ~2 min
- **Tasks:** 1
- **Files modified:** 1

## Accomplishments
- `apps/native/src-tauri/Cargo.toml`: `tauri = { version = "2", features = [] }` → `tauri = { version = "2", features = ["devtools"] }`
- No other dependency or file touched

## Task Commits

1. **Task 1: Enable devtools feature** - `f167a25` (fix)

## Files Created/Modified
- `apps/native/src-tauri/Cargo.toml` - `tauri` dependency now has `features = ["devtools"]`

## Decisions Made
- Enabling devtools for a release build is unusual (normally reserved for debug builds), but necessary here since the bug being investigated is specific to the release build's runtime behavior
- Flagged as temporary — should be reverted (back to `features = []`) once the actual root cause of the login connection failure is found and fixed, so release builds don't ship with DevTools enabled long-term

## Deviations from Plan
None — single-line change applied exactly as planned.

## Issues Encountered

## User Setup Required
- Rebuild the Tauri app (`npm run tauri build` from `apps/native`, or `cargo build --release` from `apps/native/src-tauri`) to pick up this change
- Run the rebuilt app, right-click → Inspect (or press F12) to open DevTools, reproduce the login failure, and check the Console/Network tab for the actual browser-level error (CSP violation, CORS error, net:: error code, etc.)
