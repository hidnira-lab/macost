# Quick Task 260709-2b3: Remove tipe_transaksi from frontend transaction payloads Summary

**Removed the server-derived `tipe_transaksi` field from the frontend's outgoing transaction request payloads (create + update), matching CLAUDE.md's server-side source-labeling rule -- one of the three sites had been silently sending a hardcoded, wrong-case `'pengeluaran'` on every transaction.**

## Performance

- **Duration:** ~15 min
- **Tasks:** 1
- **Files modified:** 3

## Accomplishments

- `apps/web/lib/api/types.ts`: removed `tipe_transaksi: string;` from `TransactionCreateRequest`. `TransactionUpdateRequest` is a type alias of it (`type TransactionUpdateRequest = TransactionCreateRequest`), so it's covered by the same edit -- no second field to remove. Response types (`Transaction`, which legitimately carries server-computed `tipe_transaksi`/`source_label`) left untouched.
- `apps/web/components/TransactionForm.tsx`: removed the `tipe_transaksi: selectedCategory?.tipe ?? 'Pengeluaran',` payload line. Also removed the now-unused `selectedCategory` local (its only consumer was that line) to keep lint clean, and tightened the comment above it to state the field is never sent by the frontend at all (not just "auto-derived server-side").
- `apps/web/app/transactions/new/page.tsx`: removed the hardcoded `tipe_transaksi: 'pengeluaran',` line from the POST payload -- this was live wrong data (always lowercase `'pengeluaran'` regardless of actual transaction type, and mismatched the backend's real `"Pemasukan"`/`"Pengeluaran"` casing), harmless only because the backend already ignores the field.
- Confirmed via `grep -rn "tipe_transaksi" apps/web` that all remaining matches are legitimate: reads of `trx.tipe_transaksi` off fetched transaction objects in `app/home/page.tsx` and `app/transactions/page.tsx`, the response-type field in `types.ts`, and the tightened comment -- no other file relied on the removed request field.
