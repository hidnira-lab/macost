Set `variable: "--font-body"` and `display: "swap"`.

Update the `<html>` element's `className` template string to use `neulis.variable` and `helvetica.variable` in place of `geistSans.variable` / `geistMono.variable`, keeping the existing `h-full antialiased` classes.

In apps/web/app/globals.css, inside the existing `@theme inline { ... }` block: remove the `--font-sans: var(--font-geist-sans);` and `--font-mono: var(--font-geist-mono);` lines (those variables no longer exist after Task 2's layout.tsx change) and add two new lines: `--font-display: var(--font-display);` and `--font-body: var(--font-body);`. Reusing the same name on both sides of each line is intentional and correct: `@theme inline` inlines the literal `var(...)` text directly into each generated Tailwind utility (e.g. `.font-display { font-family: var(--font-display); }`) at build time instead of emitting its own persistent `--font-display` custom property at `:root` — so the generated utility resolves against the real runtime `--font-display` custom property that `neulis.variable`'s class attaches to the `<html>` element, with no circularity.

Update the `body { ... }` rule's `font-family` declaration to read `var(--font-body), Arial, Helvetica, sans-serif` (was hardcoded `Arial, Helvetica, sans-serif`) — keep the hardcoded fallback chain after the variable for the brief pre-load/no-JS window.
