  </behavior>
  <action>
Implement `rank_goals(goals: list[dict], weights: dict, strategy: str = "quick_win") -> list[dict]` in `backend/services/saw_engine.py`:
1. `n = len(goals)`; if `n == 0` return `[]`; if `n == 1` return `[{**goals[0], "rank": 1}]` (no normalization call at all).
2. Define module-level constant `STRATEGY_MULTIPLIERS` holding the exact TC-01 multiplier table for both `quick_win` and `importance_first` per criterion (`personal_importance`, `progress_gap`, `saving_capacity`, `urgency`, `target_amount`).
3. Compute `effective_weights` = `{k: weights[k] * STRATEGY_MULTIPLIERS[strategy][k] for k in weights}`, then renormalize by dividing each by `sum(effective_weights.values())` so they sum to 1.0 — this is internal-only; the `weights` parameter itself and anything returned to callers must never reflect this multiplication (per TC-01's literal reading of D-05: only the stored/displayed weights stay unchanged, not the internal computation).
4. For each of the 5 criteria, extract raw values across all goals (`skor_keinginan` for `personal_importance`; `(nominal_target - nominal_terkumpul) / nominal_target` for `progress_gap`; the pre-computed `saving_capacity_raw` value already present on each goal dict — TC-03's ratio is computed by the caller in `goal_service.py`, not here, since it requires a DB query for `avg_monthly_side_income`; `skor_kepentingan` for `urgency`; `nominal_target` for `target_amount` — a BENEFIT per TC-04, use `normalize_benefit`, not `normalize_cost`), normalize each (benefit criteria: `personal_importance`, `saving_capacity`, `urgency`, `target_amount`; cost criteria: `progress_gap` only), multiply by `effective_weights`, sum per goal.
5. Sort by weighted score descending, tie-break by `created_at` ascending (oldest first) — this is a genuine, mathematically correct tie when all normalized criteria are identical, not a bug (per RESEARCH.md's tie-break rationale).
6. Assign `rank` starting at 1 in sorted order; return a new list of goal dicts (do not mutate the input list) each with a `rank` key added.

Write the RED-phase tests first covering every case in `<behavior>` (use `sample_goals`/`sample_weights` fixtures from `conftest.py`, adding a `saving_capacity_raw`/`created_at` field to the fixture goals as needed for this test file specifically), confirm all fail, commit `test(02-04): add failing tests for rank_goals edge cases and strategy toggle`. Implement to GREEN, commit `feat(02-04): implement rank_goals with TC-01 strategy re-weighting`. Refactor only if an obvious cleanup exists, commit `refactor(02-04): ...` only if changes were made.
