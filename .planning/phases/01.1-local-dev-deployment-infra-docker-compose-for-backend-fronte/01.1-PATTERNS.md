## Shared Patterns

### Env var naming (load-bearing, applies to `backend/.env.example`, `apps/web/.env.example`, `render.yaml`, `docker-compose.yml` env_file references)
**Source:** `backend/core/supabase.py`, `backend/dependencies/auth.py`, `apps/web/lib/api/client.ts`
**Apply to:** every file in this phase that names an env var
**Rule:** Use `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_JWT_SECRET`, `NEXT_PUBLIC_API_BASE_URL`, `NEXT_PUBLIC_USE_MOCK` â€” never the illustrative CONTEXT.md D-04 names (`SUPABASE_SERVICE_KEY`, `JWT_SECRET`, `NEXT_PUBLIC_API_URL`, `USE_MOCK`).

### Backend import namespace (load-bearing, applies to `backend/Dockerfile`, `docker-compose.yml`, `render.yaml`)
**Source:** `backend/main.py:4` â€” `from backend.routers import auth, wallets`
**Apply to:** any process/container invocation of the backend
**Rule:** CWD/build-context/rootDir must be repo root; the run command is always `uvicorn backend.main:app`, never `uvicorn main:app` from inside `backend/`.

### CORS enforcement point
**Source:** `backend/main.py` lines 8-20 (existing `CORSMiddleware` registration, before routers are included at lines 22-23)
**Apply to:** `backend/main.py` edit task (INFRA-06) â€” extend `allow_origins`, don't replace the middleware pattern.

### `.env*` gitignore negation pattern
**Source:** root `.gitignore` (already correctly scoped: `.env`, `.env.local`, `.env.*.local` â€” NOT a blanket `.env*`, so root-level `.env.example` files would already be trackable) vs. `apps/web/.gitignore` (has the broader blanket `.env*` that needs the `!.env.example` fix)
**Apply to:** `apps/web/.gitignore` edit only â€” `backend/.env.example` is not blocked by any existing `.gitignore` rule (no `backend/.gitignore` exists; root `.gitignore`'s `.env`/`.env.local`/`.env.*.local` patterns don't match `backend/.env.example`), so no gitignore fix is needed for the backend side.
