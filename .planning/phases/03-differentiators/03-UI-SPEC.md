# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** The five differentiator surfaces introduced by Phase 3 — (1) Receipt Scan (SCAN-01/02/03), (2) E-Statement PDF Import (ESTAT-01/02/03), (3) AI Financial Insights (AIINS-01/02/03), (4) SAW Weight Customization (SAW-04/05), (5) Quick Access Panel (QAP-01). This contract covers ONLY the new user-facing surfaces these features add; the base app chrome, tokens, and shared components are inherited unchanged from the approved Phase 2 contract.

**Design system inheritance:** This phase does NOT introduce a new design language. It inherits the light-theme "Palette A" system locked and Figma-verified in `.planning/phases/02-core-product-loop/02-UI-SPEC.md` (approved 2026-07-05, Figma detail finalized 2026-07-06). All spacing, typography, and color tokens below are carried forward verbatim from that approved contract and confirmed live in the codebase (`apps/web/app/ai/page.tsx`, `apps/web/app/home/page.tsx` already render these exact tokens). Only Phase-3-specific additions and deltas are called out.

**Design source of truth:** Per `CLAUDE.md` ("Sumber Desain UI"), Figma (Stitch → Figma refine) is canonical for exact per-frame layout/visuals. Per CONTEXT.md "Claude's Discretion", the precise layout of each new surface is requested from Figma frame-by-frame **during implementation** (not pre-pulled here). The relevant Phase 3 Figma frames were explicitly listed as "not yet pulled" in the Phase 2 spec — see "Figma Reference" below. This contract locks the interaction contracts, states, timing, and copy that are most likely to cause execution drift if left ambiguous; per-pixel layout comes from Figma at build time.

## Spacing Scale

Inherited verbatim from the approved Phase 2 contract (multiples of 4), confirmed against live usage in `ai/page.tsx` (`px-4`, `mb-5`, `mt-4`, `gap`, `rounded-xl`, `h-16`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, micro-adjustments |
| sm | 8px | Compact element spacing (label-to-input gap, checkbox-to-label gap) |
| md | 16px | Default element spacing (card internal padding `px-4`, form field gaps, review-row padding) |
| lg | 24px | Section padding, page horizontal padding (`px-6`) |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom spacing |

**Component radii/heights (inherited):** standard cards `rounded-xl` (12px); hero/priority/modal-top cards `rounded-3xl` (24px); progress-bar track 8–12px `rounded-full`; icon chips circular or 8–16px rounded-square on a ~10–20%-opacity tint of the icon's accent.

Exceptions (Phase 3-specific):
- **44×44px minimum touch target** for every icon-only control introduced this phase — e.g. per-row checkbox toggles in the e-statement review table, the insight-card link chevron, the SAW slider/stepper handles, and QAP tile tap areas. Every icon-only control MUST carry an `aria-label` matching its text-equivalent action (icons alone are not an accessible label). This continues the Phase 2 rule.
- No new sub-4px values. The `mt-0.5` (2px) text-baseline nudge already present in the codebase remains the only permitted micro-offset.

## Typography

Inherited target type scale from the approved Phase 2 contract (light theme). No new roles are introduced by Phase 3.

| Role | Size | Font / Weight | Usage (Phase 3 surfaces) |
|------|------|---------------|--------------------------|
| Hero stat / Display | 28–32px | Bricolage Grotesque ExtraBold | QAP balance-summary figure; brand wordmark "Macost" in the reused page header |
| Page H1 | 24px | Bricolage Grotesque ExtraBold | Page titles ("Scan Struk", "Import E-Statement", "AI Assistant", "Prioritas Goal") |
| Section heading | 18–20px | Inter Semi Bold | Section headings within a page (e.g. "Hasil Ekstraksi", "Transaksi Ditemukan", "Bobot Kriteria") |
| Body / card title | 16px | Inter Regular (body) / Inter Semi Bold (card title) | Insight card body copy; review-form field values; e-statement row merchant/description; SAW criterion name |
| Secondary | 14px | Inter Medium | Meta text, review-row category/date, insight timestamps, helper text, checkbox labels |
| Label / badge | 12px | Inter Bold | Action-verb badges, small-caps labels ("PILIH SEMUA", duplicate flag), % weight readouts, import-result counts |

Weight set: Inter Regular / Medium / Semi Bold / Bold for UI, plus Bricolage Grotesque ExtraBold reserved exclusively for wordmark, page H1s, and hero stat numbers. Body line-height 1.5; heading line-height 1.2.

> **Note (checker FLAG, non-blocking):** This inherited scale declares 6 size tiers and 5 weights, which exceeds the nominal single-screen caps (≤4 sizes, ≤2 weights). This is a **conscious carry-forward**, not drift: the scale is inherited verbatim from the approved, Figma-verified Phase 2 contract and is already shipped live (`ai/page.tsx`, `home/page.tsx`), so an additive phase cannot reduce it without regressing existing screens. Tiers are cleanly role-separated (display / h1 / section / body / secondary / label), avoiding the visual noise the cap guards against. No single new Phase-3 surface should use all tiers at once.

|------|-------|-------|

| Surface / cards (secondary, ~30%) | `#ffffff` fill + `rgba(30,30,30,0.15)` border | Insight cards, review-form card, e-statement review container, SAW criterion cards, QAP tiles |

| Subtle fills | `rgba(30,30,30,0.05)`–`rgba(30,30,30,0.08)` | Icon-chip backgrounds, progress/slider track backgrounds, disabled/idle dropzone fill |
| Accent blue (~10%) | `#298dff` (→ `#065fc5` gradient on filled buttons/FABs; `#a8c8ff` on progress fills) | Primary actions, links, active nav, input focus — see reserved-for list |
| Accent orange | `#ff8929` (→ `#ffb787` gradient on fills) | Goals / priority / positive emphasis — SAW rank-#1 highlight in the re-rank preview, QAP top-goal progress fill, "Alokasikan" action-verb badge |

**Accent blue (`#298dff`) reserved for — explicit list (never "all interactive elements"):**
- Primary filled CTAs: "Pilih File" / "Upload Struk" (scan), "Pilih PDF" / "Import Terpilih" (e-statement), "Simpan Transaksi" (scan review save), "Perbarui insight" (AI refresh), "Simpan Bobot" / "Apply Weights" (SAW editor)
- The circular FAB / quick-add affordance already present on Home (unchanged)
- Input focus border on all new form inputs (scan review fields, SAW numeric weight inputs)
- Insight-card link/chevron to a goal or category (AIINS-02 link target)
- Active/selected state of any segmented control reused this phase
- The upload dropzone active/drag-over border and the extraction/insight loading spinner

**Accent orange (`#ff8929`) — secondary emphasis, reserved for:**
- The SAW rank-#1 goal in the live re-rank preview (mirrors the "Priority #1" differentiated treatment from Phase 2)
- The QAP "top active goal" tile progress fill (positive/goal emphasis)

**Destructive/error tokens reserved for:** scan extraction failure/timeout banner (SCAN-03), e-statement parse failure banner, AI insight timeout fallback banner (AIINS-03), and the "Kurangi" action-verb badge (reduce-spending, cautionary). **Note:** the e-statement possible-duplicate flag is a *warning/caution*, NOT destructive — render it in a muted/amber-neutral treatment (e.g. `rgba(30,30,30,0.65)` text + a subtle badge), not in the `#ba1a1a` destructive red, so it does not read as an error.

**Action-verb badge color map (locked — removes executor ambiguity for AIINS-02):**

- `Kurangi` → destructive `#ba1a1a` on `#ffdad6` tint (cautionary reduce)

## Interaction & State Contracts (Phase 3-specific)

These sequences are locked by CONTEXT.md decisions (D-01..D-06) and the locked API contract. The executor should NOT need to re-derive them from a Figma frame; Figma governs pixel layout only.

### Receipt Scan (SCAN-01/02/03, D-01)
1. **Input is file-upload only** — a file picker accepting JPG/PNG. Do NOT build a webcam/`getUserMedia` capture UI (MVP is Web + Tauri Desktop, no mobile camera). The dropzone/button may say "Pilih File Struk" or support drag-and-drop, but there is no live-camera affordance.
2. On file select, send `multipart/form-data` field `image` to `POST /api/transactions/scan-receipt` (contract locked — do not rename the field or change the shape). Show the blocking "Mengekstrak data struk…" loading state.
3. **Hard 10s timeout** (per D-01/SCAN-03). On success (`extracted: true`), transition into an **editable review form** pre-filled with `merchant`, `nominal`, `tanggal_transaksi`, `items` (optional), and `suggested_category_id`. **Never auto-save** (SCAN-02) — the user must review/correct and press "Simpan Transaksi".
4. The review form reuses the Phase 2 transaction-form field styling and the "3 required fields + pre-filled date" contract. Frontend still never sends `tipe_transaksi`/`source_label` — the backend derives them from the chosen `kategori_id` on save (extracted `suggested_category_id` pre-selects the category but stays user-editable).
5. On failure or timeout (`extracted: false`, or >10s): show the destructive-token fallback banner with `error_message` and route the user to **manual input** (SCAN-03). **No auto-retry** (D-01).

### E-Statement PDF Import (ESTAT-01/02/03, D-02)
1. Upload a PDF via `multipart/form-data` field `file` to `POST /api/transactions/upload-statement` (contract locked). Show "Membaca statement…" loading.
2. Response `extracted_transactions[]` (each with `temp_id`, `tanggal_transaksi`, description, `nominal`, `is_possible_duplicate`, `suggested_category_id`) renders as a **review table/list**, one selectable row each.
3. **Default selection (D-02):** every row is checkbox-checked by default **EXCEPT** rows with `is_possible_duplicate: true`, which render **unchecked** and carry the cautionary "Mungkin duplikat" badge. The user can freely toggle any row (check a flagged duplicate, uncheck a normal one) before confirming.
4. Provide a "Pilih semua / Hapus pilihan" affordance and a live selected-count that feeds the CTA label "Import Terpilih ({n})".
5. On confirm, batch-import only the checked rows; then report the result inline/toast: "{imported_count} transaksi diimpor, {skipped_count} dilewati." (ESTAT-03). Nothing is saved before this explicit confirm (suggest-and-confirm extended to batch import).

### AI Financial Insights (AIINS-01/02/03, D-03/D-04)
1. **Generation timing (D-04):** insights are generated **on page open** of `/ai` (server-side LLM call) plus a manual "Perbarui insight" refresh button. Wire the existing `apps/web/app/ai/page.tsx` shell to `GET /api/ai-insight` (reuse its header/chrome; do not rebuild the page).
2. **One-way only** — this is an insight feed, NOT a chat. Do not add a text input, message composer, or conversational affordance (AIAGENT-01 is explicitly post-MVP; do not build it).
3. Each insight card renders (per the D-03 extended contract): the Bahasa-Indonesia insight text, an **action-verb badge** (`Alokasikan` | `Kurangi` | `Pertimbangkan` — fixed vocabulary, colored per the badge map above), and a **link chevron** to the related goal (`related_goal_id`) or category (`related_category_id`). Exactly one link target per insight; if both are null the card shows no chevron.
4. **15s hard timeout is server-side** (D-04). On timeout/failure the endpoint returns `insight_available: false` + `fallback_message`; the UI shows the fallback banner and the "Buka Goals" CTA routing to the **Goals page** (locked target, AIINS-03). The frontend does not implement its own competing timeout/spinner-cutoff.

> **⚠ Execution prerequisite (D-03):** the `action_verb` + `related_category_id` additions to `GET /api/ai-insight` change `API_CONTRACT.md`, which per project rules MUST be communicated to and signed off by all 4 team members BEFORE the insight feature is implemented. The planner/executor must treat the contract update + team sign-off as a prerequisite task, not silently alter the shape. This UI-SPEC assumes the extended shape; it does not authorize skipping the sign-off.

### SAW Weight Customization (SAW-04/05, D-05)
1. **Manual numeric entry** for the five criteria (`personal_importance`, `progress_gap`, `saving_capacity`, `urgency`, `target_amount`) — each in its own card with a colored dot + label + editable value. (A visual slider/segment bar may accompany the numeric field per Figma frame `156:824`, but the numeric field is the source of truth per D-05.)
2. **Live re-rank preview:** as weights change, show the resulting goal ranking updating (drives `saw_engine.py` with candidate weights via the existing ranked `GET /api/goals` mechanic). The SAW rank-#1 goal in the preview gets the accent-orange rank-#1 emphasis.
3. **Sum validation uses ±0.002 tolerance** (D-05), NOT ±0.001 — the locked research defaults sum to 0.999 and would fail a ±0.001 check. Show the "Total bobot harus 100%. Saat ini {sum}%." inline error and disable "Simpan Bobot" until the sum is within tolerance.
4. **"Reset ke default" button** (SAW-05) restores the n=62 research weights (personal_importance 22.5%, progress_gap 21.9%, saving_capacity 21.5%, urgency 17.8%, target_amount 16.2%) — NEVER the illustrative 25/20/20/15/20 placeholders shown in Figma frame `156:824`. Light inline confirm before applying.

> **⚠ Execution prerequisite (D-05):** update SAW-04 text in `.planning/REQUIREMENTS.md` from `±0.001` to `±0.002` so the requirement matches the validator tolerance and the 0.999-sum defaults. Prerequisite task, not scope creep.

### Quick Access Panel (QAP-01, D-06)
1. QAP is a **new component mounted at the TOP of the existing `apps/web/app/home/page.tsx`** (already wired to dashboard + goals APIs). Reuse the Home page's existing fetches — **do NOT add new endpoints** for QAP.
2. **Exactly 4 shortcuts, each one-tap:** (1) Add Transaction, (2) Scan Receipt, (3) Top active goal — name + progress % using the **SAW rank-1 goal**, (4) Balance summary. Not 3, not 5.
3. Keep it clean/uncrowded with large, thumb-reachable targets (≥44px hit area each). The top-goal tile uses the accent-orange progress emphasis; the balance figure uses the Bricolage Grotesque hero-stat treatment. Exact grid/layout is Figma-driven; the 4-item count and their identities are locked.
4. If there is no active goal, the top-goal tile shows "Belum ada goal" and routes to goal creation/onboarding rather than rendering an empty progress bar.
