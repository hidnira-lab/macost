## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| FastAPI -> Gemini API | Server-to-server call forwarding aggregated (but not raw-file) user financial data as a text prompt |
| insight_service aggregation -> transaksi/goal tables | Must scope exclusively to the requesting user |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-3-18 | Information Disclosure | Insight aggregation query missing id_pengguna scope, leaking cross-user financial data into a prompt | critical | mitigate | `generate_user_insights` reuses `fetch_and_rank_goals(user_id)` (already IDOR-safe) and a transaction query scoped identically to `list_transactions`'s `.eq("id_pengguna", ...)` pattern — verified by a dedicated IDOR test case |
| T-3-19 | Tampering | LLM output violating the related_goal_id/related_category_id invariant, breaking the AIINS-02 link requirement | medium | mitigate | `InsightItem`'s `model_validator` (03-01) rejects any Gemini response with both fields null at the model-parsing layer inside `gemini_service.generate_insight` — a violating response never reaches the router as a "successful" result |
| T-3-20 | Denial of Service / Availability | Insight generation appearing to hang past 15s with no user feedback | high | mitigate | AIINS-03's 15s hard timeout is enforced inside `gemini_service.generate_insight` (03-01), not re-implemented here; `GET /api/ai-insight` always returns promptly with either real insights or the fallback shape — never blocks past that ceiling |
| T-3-21 | Elevation of Privilege | Interactive/agentic behavior creeping into the one-way insight feature | low | accept | AIAGENT-01 (chat/tool-calling) is explicitly post-MVP and out of scope; Task 2 explicitly forbids adding any input/composer UI — risk is a scope-discipline concern, not a technical vulnerability, and is fully mitigated by the plan's own scope boundary |

</threat_model>
